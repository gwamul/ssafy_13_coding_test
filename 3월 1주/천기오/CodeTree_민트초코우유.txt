문제 전략
1. 역시나 문제 요구사항을 '그대로' 수행하면 되는 문제
2. 자료구조를 고민해 봤는데, 신앙심과 신앙 음식은 2차원 배열로 저장했다.
3. 전파 구조를 읽어보니 음식은 비트로 표현하면 깔끔하게 풀리겠다는 생각이 들었다. 
  민트001 초코010 우유100 로 표현한다면 약한 전파시 |(or) 연산만 하면 되고, 마지막 신앙심 계산시에도 쉽게 할 수 있다.
4. 그룹의 경우 클래스로 정의하는게 낫겠다 싶어서 class를 정의했다. 대표자의 좌표를 저장하고, 그룹에 속한 학생들 좌표를 리스트로 가진다.
class Group {
	int px, py;
	int type = -1;
	List<int[]> students = new ArrayList<>();
}
5. 자료를 잘 정리했다면 이제 문제 요구대로 차례대로 풀이한다.


1 - 아침
아침은 할게 없다. 모든 신앙심 +1만 한다.

2 - 점심
선호 음식이 같은 사람끼리 bfs로 묶는다. bfs에서 리스트에 넣기 전에 대표자를 검사한다. 신앙심이 큰 순서 - 행/열 값이 작은 값으로 업데이트해서 최종 대표를 선정한다.
그룹이 생성되면 그룹들을 순회하여 대표자에게 신봉자들의 수만큼 + 하고, 신봉자들은 -1 한다.

3 - 저녁
전파를 실행한다. 전파 순서를 맞추기 위해 우선 그룹들을 정렬한다.
정렬 1. 선호 음식의 가짓수가 적은 순서(비트카운트로 판별한다)
정렬 2. 대표자의 신앙심이 큰 순서
정렬 3. 행/열 값이 작은 순서

이 정렬 순서를 잘 맞추는게 이 문제 핵심인거 같다.
정렬 후 그룹에서 순서대로 전파를 실행한다. 여기서 주의할 점은 다른 그룹에서 전파당한 그룹은 스킵해야한다는 점.
전파 방식은 철저히 문제 정의에 따른다.

4 - 정산
음식을 비트로 표현했기 때문에 정산은 매우 간편하다.
// 총계
int[] result = new int[8];
for (int i = 0; i < n; i++) {
  for (int j = 0; j < n; j++) {
    result[types[i][j]] += scores[i][j];
  }
}
111(7) 이 민트초코우유이고 001(1)이 민트다. 저렇게만 작성하면 자연스럽게 정산이 된다.
마지막 출력 순서가 비트랑 맞지 않아 골치 아팠는데, 이런경우 배열로 순서를 정의하면 좋다.
